# 📚 Study Tracker

스터디 시간을 추적하고 관리하는 반응형 웹 애플리케이션입니다. 바닐라 자바스크립트로 구현되었으며, 로컬 스토리지를 사용하여 데이터를 저장합니다.

## ✨ 주요 기능

### 1. 스터디 관리 (Study 페이지)

#### 스터디 등록
- 카테고리와 스터디명으로 스터디를 등록할 수 있습니다
- 등록된 스터디는 그리드 형식으로 표시됩니다

#### 스터디 시간 입력
- 스터디 카드를 클릭하면 시간 입력 팝업이 나타납니다
- 다음 정보를 입력할 수 있습니다:
  - 날짜 (기본값: 오늘 날짜)
  - 시간대: 출근길, 아침, 점심, 퇴근길, 퇴근후, 기타
  - 수행 시간 (분 단위)

#### 스터디 수정
- 스터디 카드를 길게 누르면(500ms) 수정 팝업이 나타납니다
- 카테고리와 스터디명을 수정할 수 있습니다
- 마우스와 터치 이벤트 모두 지원합니다

### 2. 챌린지 (Challenge 페이지)

#### 월별 뷰
- Challenge 메뉴 접속 시 월별 합계가 표시됩니다
- 1월부터 12월까지 각 월의 합계를 한눈에 확인할 수 있습니다
- 각 월별로 다음 정보를 표시합니다:
  - 시간대별 합계 (출근길, 아침, 점심, 퇴근길, 퇴근후, 기타)
  - 월별 총합계
  - 해당 월에 수행한 스터디 목록

#### 일별 뷰
- 월을 클릭하면 해당 월의 일별 상세 정보가 표시됩니다
- 1일부터 마지막 날까지 모든 일자를 표로 표시합니다
- 각 일자별로 다음 정보를 표시합니다:
  - 날짜 (요일 포함)
  - 시간대별 합계
  - 일일 총합계
  - 해당 일자에 수행한 스터디 목록
  - 메모

#### 년도 선택
- 상단의 좌우 화살표 버튼으로 년도를 변경할 수 있습니다
- 선택한 년도의 데이터를 확인할 수 있습니다

#### 기록 수정
- 시간 셀, 합계 셀, 스터디 목록 셀을 클릭하면 수정 모달이 열립니다
- 수정 모달에서 다음 작업을 할 수 있습니다:
  - 기존 기록의 수행 시간 수정
  - 기록 삭제
  - 새로운 기록 추가
  - 메모 수정

#### 메모 기능
- 일별 뷰에서 각 일자별로 메모를 입력할 수 있습니다
- 메모 셀을 클릭하면 입력 모드로 전환됩니다
- Enter 키를 누르거나 포커스를 잃으면 자동으로 저장됩니다
- 메모가 없을 때는 "메모 입력" 플레이스홀더가 표시됩니다

### 3. 데이터 저장
- 모든 데이터는 브라우저의 로컬 스토리지에 저장됩니다
- 페이지를 새로고침해도 데이터가 유지됩니다
- 나중에 백엔드와 연결할 수 있도록 구조화되어 있습니다

## 🚀 시작하기

### 설치
별도의 설치 과정이 필요 없습니다. 브라우저에서 `index.html` 파일을 열면 바로 사용할 수 있습니다.

### 사용 방법

1. **스터디 등록**
   - Study 페이지에서 "스터디 등록" 버튼을 클릭합니다
   - 카테고리와 스터디명을 입력하고 등록합니다

2. **시간 입력**
   - 등록된 스터디 카드를 클릭합니다
   - 날짜, 시간대, 수행 시간을 입력하고 저장합니다

3. **스터디 수정**
   - 스터디 카드를 길게 누릅니다 (500ms 이상)
   - 수정 팝업에서 정보를 수정합니다

4. **챌린지 확인**
   - Challenge 메뉴로 이동합니다
   - 월별 합계를 확인하거나, 월을 클릭하여 일별 상세를 확인합니다
   - 년도를 변경하여 다른 년도의 데이터를 확인할 수 있습니다

5. **기록 수정**
   - 일별 뷰에서 시간 셀, 합계 셀, 또는 스터디 목록 셀을 클릭합니다
   - 수정 모달에서 기록을 수정, 삭제, 또는 추가합니다

6. **메모 입력**
   - 일별 뷰에서 메모 셀을 클릭합니다
   - 메모를 입력하고 Enter 키를 누르거나 다른 곳을 클릭합니다

## 🛠️ 기술 스택

- **HTML5**: 마크업 구조
- **CSS3**: 스타일링 및 반응형 디자인
- **바닐라 자바스크립트**: 모든 기능 구현
- **로컬 스토리지**: 데이터 저장

## 🧪 테스트

프로젝트에는 단위 테스트와 통합 테스트가 포함되어 있습니다.

### 테스트 실행 방법

1. **브라우저에서 실행**:
   - `index.html`에서 `test.js` 스크립트 태그의 주석을 해제
   - 페이지를 새로고침하면 자동으로 테스트 실행
   - 개발자 도구 콘솔에서 결과 확인

2. **Node.js에서 실행** (선택사항):
   ```bash
   node test.js
   ```

### 테스트 범위

- 데이터 관리 함수 (추가, 수정, 삭제)
- 유틸리티 함수 (날짜 포맷팅, HTML 이스케이프 등)
- 데이터 처리 함수 (필터링, 그룹화, 합계 계산)
- 에러 처리 및 입력 검증
